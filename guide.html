🚀 Quick Start 01

Four steps to get the system running end-to-end on GitHub Pages.

  1. Push this repo to GitHub. Create a new repo and push all files.
  2. Add your API key as a GitHub Secret. Go to Settings → Secrets → Actions. Add ANTHROPIC_API_KEY (or OPENAI_API_KEY / GOOGLE_API_KEY depending on your chosen provider).
  3. Enable GitHub Pages. Go to Settings → Pages → Source: GitHub Actions.
  4. Trigger your first run. Go to Actions → Intelligence Pipeline → Run workflow. Or go to config.html and click "Generate Report Now".
✓ After the first run completes (~10 min), your reports will be live at https://[username].github.io/[repo-name]/
🔑 GitHub Secrets 02

Secrets are API keys stored securely in GitHub — never in code or committed files. Only the key for your active provider is required.

Secret NameRequired forWhere to get it
ANTHROPIC_API_KEYClaude (Anthropic)console.anthropic.com → API Keys
OPENAI_API_KEYGPT (OpenAI)platform.openai.com → API Keys
GOOGLE_API_KEYGemini (Google)aistudio.google.com → Get API Key

How to add a secret

  1. Go to your GitHub repo → Settings
  2. Click Secrets and Variables → Actions
  3. Click New repository secret
  4. Name: ANTHROPIC_API_KEY (or the relevant key) · Value: your actual key
⚠ Never paste API keys into config files, code, or HTML. They'll be committed to git and exposed publicly. Always use GitHub Secrets.

For local testing

When running scripts locally, export keys in your terminal before running:

export ANTHROPIC_API_KEY=sk-ant-...
python scripts/run_pipeline.py

How to switch provider

Open config/providers.yaml in the config editor and change active_provider:

active_provider: openai   # was: anthropic

Then save. The next pipeline run will use GPT. Make sure OPENAI_API_KEY is in GitHub Secrets.

How to change model

Find the provider block and update its model field:

anthropic:
  model: claude-sonnet-4-6   # changed from claude-opus-4-7
ℹ Switching to a lighter model (Sonnet, Haiku, gpt-4o-mini, gemini-flash) reduces cost and increases speed. Use for high-frequency runs. Use the flagship models for the best analysis depth.

📡 News Feeds 04

File: config/feeds.yaml — controls where news comes from.

Each feed entry has a url (RSS/Atom), a category (drives the report color theme), and a trust score (0.5–1.5 multiplier on the intelligence score).

Add a new feed

feeds:
  - url: https://example.com/rss
    category: technology       # ai | technology | geopolitics | macro | india | energy | finance | science | general
    trust: 1.0                 # 0.5 = low trust, 1.0 = neutral, 1.5 = high trust
    name: "Example Source"     # display name

Category → Report Color

The category you set on a feed determines the accent color of every report from that source. Themes are automatic — no manual work needed.

⏱ Schedule & Limits 05

File: config/schedules.yaml — controls when and how much the system runs.

🕐 primary_schedule
Cron expression for main runs. Default: "0 6,18 * * *" (6am + 6pm UTC).
Note: also update the cron in .github/workflows/intelligence.yml to match.
📊 stories_per_run
How many top-ranked articles get deep AI analysis per run. Default: 5. Higher = richer output, more API cost.
🪙 max_tokens_per_analysis
Max AI output tokens per story. Default: 6000. Higher = deeper analysis, more cost. Range: 3000–10000.
🕰 fetch_window_hours
How far back to look for news on each run. Default: 24 hours. Set to 4 for every-3-hour runs.

Common schedule presets

primary_schedule: "0 8 * * *"      # Once daily at 8am UTC
primary_schedule: "0 6,14,22 * * *" # Three times daily
primary_schedule: "0 */4 * * *"     # Every 4 hours
primary_schedule: "0 8 * * 1-5"     # Weekdays only
⚠ The schedule in schedules.yaml is informational/used by scripts. The actual GitHub Actions trigger is the cron in .github/workflows/intelligence.yml — update both to stay in sync.
⚖ Topic Weights 06

File: config/topic_weights.json — controls which news topics score higher in ranking.

Each topic has a weight (how much it boosts the score) and a list of keywords (matched against article title + summary).

{
  "topics": {
    "artificial_intelligence": {
      "weight": 2.0,
      "keywords": ["AI", "machine learning", "LLM", "GPT", "neural network"]
    },
    "your_new_topic": {
      "weight": 1.5,
      "keywords": ["keyword1", "keyword2", "keyword3"]
    }
  }
}

Weight guide

3.0Extremely high priority — almost always selected
2.0High priority — strongly preferred
1.5Above average
1.0Neutral — default
0.5Below average — deprioritized
👤 Entity Tracking 07

File: config/entity_priorities.json — specific people, companies, countries, and technologies that automatically boost a story's score.

Priority 1 entities add +15 points each. Priority 2 add +8. Priority 3 add minor boosts.

{
  "people": {
    "priority_1": ["Sam Altman", "Narendra Modi", "Jensen Huang"],
    "priority_2": ["Sundar Pichai", "Elon Musk"]
  },
  "companies": {
    "priority_1": ["OpenAI", "NVIDIA", "Reliance Industries"],
    "priority_2": ["Google", "Microsoft", "Apple"]
  },
  "countries": {
    "priority_1": ["India", "China", "United States"],
    "priority_2": ["Taiwan", "Russia", "EU"]
  }
}
ℹ Entity matching is case-insensitive substring matching. "NVIDIA" matches "Nvidia Corporation", "NVIDIA stock", etc.
🧠 Analysis Prompt 08

File: config/analysis_prompt.md — the most powerful config file. This is the exact prompt sent to the AI for every story analysis. Nothing about the analysis is hardcoded in Python — the output structure, the analyst persona, the sections, the depth — all of it lives here.

System Prompt section
Defines the analyst's persona, expertise areas, and analytical philosophy. Change this to get a different analytical lens — e.g. "you are a geopolitical risk analyst focused on Southeast Asia" or "you are a VC investor focused on deep tech."
User Prompt Template section
The actual analysis request, including the output JSON schema. Every section in the report (executive summary, scenarios, strategic incentives, etc.) is defined here. Add or remove sections by editing the JSON schema in this file — the report HTML will adapt automatically.
Analysis Style Guidelines section
Behavioral rules: depth over breadth, contrarian lean, time horizons. Edit these to shape how the AI reasons — e.g. add "always include a startup opportunity section" or "weight regulatory risk heavily."
⚠ The output JSON schema in the prompt must stay valid JSON structure. Adding new keys is safe — the report renderer handles unknown keys gracefully. Removing required keys may cause rendering gaps.
🌍 Worldview 09

File: config/worldview.md — a persistent context document injected into every analysis. This is how the system builds compounding intelligence over time.

The worldview starts empty and gets automatically updated by the system after each analysis run — the AI extracts new insights, theses, and patterns and appends them here. You can also edit it manually.

What to put here

Think of it as a "standing brief" — what does the analyst already know going into each story?

## Active Theses
- India is in a 10-year window of demographic dividend, peaking ~2035
- AI compute is the new oil — whoever controls GPU supply controls AI

## Recurring Patterns I've Noticed
- Tech regulation always lags 5-7 years behind technology adoption

## Current Geopolitical Context
- US-China tech decoupling is accelerating; Taiwan is the flashpoint
ℹ The worldview gets more valuable over time. After 30–60 days of runs, the system has accumulated significant context and produces notably deeper analysis. Don't clear it unless starting a fresh focus area.
⚡ Ad-hoc Reports 10

Generate a report on any topic or question instantly — without waiting for the next scheduled run and without needing a news article.

From config.html (easiest)

  1. Go to config.html and connect your repository.
  2. In the "Ad-hoc Analysis" box in the sidebar, type your topic or phrase.
  3. Click ▶ Analyze This Topic. Confirm the modal.
  4. Wait ~5-10 minutes. The report appears in your GitHub Pages site.

From GitHub Actions (manual)

  1. Go to your repo → Actions → Intelligence Pipeline
  2. Click Run workflow
  3. Fill in the custom_topic field with your topic
  4. Click Run workflow

Good topic examples

Impact of AI on Indian software export industry
The geopolitics of lithium supply chains
Why central bank digital currencies are accelerating
Second-order effects of US tariffs on Vietnamese manufacturing
ℹ Ad-hoc reports skip the fetch and rank steps entirely. The AI receives your topic as-is and applies the full analysis framework from analysis_prompt.md.
🎨 Report Design 11

Report design is controlled by Python in scripts/generate_report.py. The category-contextual color system is automatic — no configuration needed.

How contextual theming works

Each report's category (set via the feed's category field) maps to a CSS hue value. A single --cat-h variable drives the entire color palette for that report — hero glow, section borders, score badges, everything.

CATEGORY_HUE = {
    "ai": 263,           # violet
    "technology": 220,   # blue
    "geopolitics": 38,   # amber
    "macro": 174,        # teal
    "india": 25,         # saffron
    "energy": 142,       # green
    "finance": 157,      # emerald
    "science": 197,      # sky
}

To add a new category color: edit CATEGORY_HUE in generate_report.py and add the matching category to your feeds.

ℹ The analysis prompt and HTML design are intentionally separated. The AI outputs structured JSON — the Python renderer turns it into beautiful HTML. To change design, edit the Python. To change analysis depth/sections, edit analysis_prompt.md.
❓ FAQ 12
How do I change the AI provider mid-stream?
Edit active_provider in providers.yaml and ensure the matching API key secret exists in GitHub. The change takes effect on the next run. Old reports are unaffected.
Why is my feed not showing up?
Check: (1) The URL is a valid RSS/Atom feed — open it in a browser, it should show XML. (2) The feed has articles published within fetch_window_hours. (3) YAML indentation is correct in feeds.yaml.
How do I make the system focus more on India?
Three levers: (1) Add India-focused feeds with category: india and trust: 1.3. (2) Increase india weight in topic_weights.json. (3) Add key India entities to priority_1 in entity_priorities.json.
How do I reduce API costs?
Three levers: (1) Switch to a lighter model in providers.yaml. (2) Reduce stories_per_run in schedules.yaml. (3) Reduce max_tokens_per_analysis. Each story costs roughly 3-8 API tokens × model price.
Can I add my own analysis sections?
Yes. Add a new key to the JSON schema in analysis_prompt.md's user prompt template. Then add a matching entry to the SECTIONS list in generate_report.py. The HTML renders any section the AI returns.
How do I reset the worldview/memory?
Delete or clear the contents of config/worldview.md and memory/ directory files via the config editor. The system will start fresh on the next run.
