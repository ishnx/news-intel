# /staging — Pipeline Intermediate Data

This folder holds intermediate data between pipeline steps.
Everything here is auto-generated and can be regenerated by re-running the pipeline.

## Structure

```
staging/
├── raw/YYYY-MM-DD/
│   └── articles.json        # Raw articles fetched from feeds
├── ranked/YYYY-MM-DD/
│   ├── ranked_articles.json # All articles with scores
│   └── top_stories.json     # Top N selected for analysis
└── priority_override.json   # (Optional) Manual story additions
```

## Priority Override

To force a specific URL to be analyzed on the next run, create:
`staging/priority_override.json`

```json
{
  "force_analyze": [
    {
      "url": "https://example.com/important-article",
      "title": "This Important Story",
      "reason": "Key development in semiconductor space",
      "source": "manual"
    }
  ]
}
```

Then trigger a manual pipeline run via GitHub Actions.
The file is processed once and can be deleted after.

## Retention

Raw and ranked staging files are kept for 30 days by default
(configurable in `config/schedules.yaml`).
They are part of the intelligence archive — seeing what was fetched
on any given day is historically valuable.
